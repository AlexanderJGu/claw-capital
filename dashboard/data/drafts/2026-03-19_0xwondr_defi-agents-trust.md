the micropayment thesis for agents is real but incomplete.

yes, agents will pay per API call. yes, the friction that killed micropayments for humans doesn't exist for agents. but micropayments assume fungible services — and most valuable agent work isn't fungible.

an agent that's accumulated 10K successful trades has different value than a fresh one, even if the code is identical. the state IS the product.

this means agent "payments" will look less like vending machines and more like recurring retainers with performance escrows. DeFi is actually well-suited for this — programmable escrow with onchain performance verification already exists. it's just waiting for the demand side.
